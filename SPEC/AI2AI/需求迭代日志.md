2. **操作方式改进**
   - 向左滑动: 角色向左移动

   - 添加 `JUMPING` 状态枚举
   - 添加跳跃物理系统:
     - 垂直速度 (`velocityY`)
     - 跳跃力度 (`JUMP_FORCE = -400`)
     - 重力加速度 (`GRAVITY = 1200`)
     - 地面Y坐标 (`groundY`)
   - 实现 `jump()` 方法触发跳跃
   - 实现 `update(deltaTime)` 方法处理重力物理
   - 落地检测和状态切换

4. **演示场景集成** (`src/scenes/BattleScene.ts`)
   - 更新角色初始化,设置地面Y坐标
   - 集成摇杆跳跃检测
   - 更新角色物理状态
   - 跳跃时显示移动动画

### 技术细节

**摇杆设计**
- 外圈半径: 60px
- 内圈半径: 21px (外圈的35%)
- 触摸范围: 120px (外圈的2倍)
- 跳跃阈值: 向上滑动超过40%

**跳跃物理**
- 跳跃力度: -400 (向上)
- 重力: 1200 (向下加速)
- 物理更新频率: 每帧根据deltaTime计算

**视觉效果**
- 发光效果: `shadowBlur` 15-50px
- 渐变填充: 径向渐变
- 呼吸动画: `sin` 函数控制缩放
- 发光强度衰减: 操作后逐渐减弱

## 2026-03-16 - Bug修复：玩家ID和视角问题

### 问题描述

好友通过链接进入房间后：
1. 两个人的名字是一样的
2. 好友进入后第一人称视角还是房主
3. 好友控制的也是房主

### 根本原因

1. **玩家ID生成问题**：
   - 原代码使用 `wx.getStorageSync('playerId') || this.generateRoomId()`
   - 在微信开发者工具中，不同模拟器实例可能共享同一个 localStorage
   - 导致两个设备使用相同的 `playerId`，查找当前玩家时总是找到第一个（房主）

2. **昵称重复问题**：
   - `wx.getUserInfo()` 在开发者工具中可能返回相同的测试用户信息
   - 没有添加唯一标识，导致昵称重复

### 修复内容

1. **玩家ID生成优化** (`src/services/RoomService.ts`)
   - 新增 `generatePlayerId()` 方法，使用时间戳+随机数确保唯一性
   - `createRoom()` 和 `joinRoom()` 不再使用缓存的 `playerId`
   - 每次都生成新的玩家ID，避免冲突

2. **昵称唯一性优化** (`src/services/RoomService.ts`)
   - 在 `getCurrentPlayer()` 中添加玩家ID后缀
   - 格式：`昵称#后4位ID`（如：`测试用户#A1B2`）
   - 确保即使获取到相同的用户信息也能区分

### 当前状态

玩家ID和视角问题已修复，每个玩家都有唯一的ID和昵称。

### 测试建议

建议在真机上测试，微信开发者工具的模拟环境可能存在以下限制：
- localStorage 可能跨实例共享
- 用户信息可能返回相同的测试数据

## 2026-03-16 - Bug修复：双人模式分享链接问题

### 问题描述

好友通过分享链接进入房间时，没有加入现有房间，而是新创建了一个房间。

### 根本原因

1. **CloudBase 初始化异步问题**：`RoomService` 构造函数中调用 `initCloudBase()` 是异步的，但没有等待完成。当 `joinRoom` 或 `createRoom` 被调用时，`collection` 可能还未初始化完成，导致数据库操作失败或行为异常。

2. **状态管理器不支持传递附加数据**：`GameStateManager.setState` 方法不支持传递附加数据，导致 `roomId` 无法正确传递到 `RoomScene`。

### 修复内容

1. **CloudBase 初始化优化** (`src/services/RoomService.ts`)
   - 添加 `initPromise` 字段，确保初始化只执行一次
   - 实现等待机制，确保初始化完成后再执行后续操作
   - 防止并发初始化导致的问题

2. **状态管理器增强** (`src/core/GameState.ts`)
   - 修改 `setState` 方法支持传递附加数据（`data` 参数）
   - 修改监听器签名支持接收附加数据
   - 添加 `getData` 方法获取当前状态数据

3. **日志增强**
   - 在 `handleLaunchOptions` 中增加详细的参数获取日志
   - 在 `joinRoom` 和 `createRoom` 中增加详细的执行流程日志
   - 便于调试和问题追踪

### 当前状态

双人模式分享链接问题已修复，好友通过链接可以正确加入房间。
